= Creating and running a migration plan

Creating and running a migration plan involves the following key steps:

. Checking the xref:Migration_plan_prerequisites_and_options[migration plan prerequisites and options] if any of the following conditions apply:

* You are migrating previously migrated virtual machines
* You are migrating virtual machines running RHEL or other Linux operating system
* You are using Ansible playbooks for premigration/postmigration tasks

. xref:Creating_a_migration_plan[Creating a migration plan] with the *Create Migration Plan* wizard in CloudForms

. Running the migration plan, the last step of the *Create Migration Plan* wizard, with one of the following options:

* *Save migration plan to run later*
** xref:Scheduling_a_saved_migration_plan[Scheduling a saved migration plan]

* *Start migration immediately*
** xref:Viewing_migration_plan_progress[Viewing a migration plan in progress]
** xref:Canceling_a_migration_plan[Canceling a migration plan in progress]

If external factors, such as network disruption, cause migration errors, see xref:Retrying_a_failed_migration_plan[].

For other migration errors, see xref:Troubleshooting[].
