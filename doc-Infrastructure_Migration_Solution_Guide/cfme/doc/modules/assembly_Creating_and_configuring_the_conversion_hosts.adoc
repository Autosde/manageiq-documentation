= Creating and configuring the conversion hosts

A _conversion host_ can be a Red Hat Virtualization host or an OpenStack Platform instance created from a conversion host appliance. `virt-v2v` and `virt-v2v-wrapper`, running on the conversion host, perform the migration process.

Creating and configuring the conversion hosts involves the following key steps:

. OpenStack Platform only: xref:Creating_OSP_conversion_hosts[Creating the OpenStack Platform conversion hosts]
. xref:Configuring_conversion_hosts_for_transformation[Configuring the conversion hosts for VDDK or SSH transformation]
. Red Hat Virtualization only: xref:Authenticating_rhv_conversion_hosts_in_cloudforms[Authenticating the Red Hat Virtualization conversion hosts]
. Validating the xref:Verifying_conversion_hosts[conversion hosts]

[NOTE]
====
If you upgrade your target environment, upgrade the conversion hosts:

* Red Hat Virtualization: Enter the following commands on the Manager machine:
+
[options="nowrap" subs="+quotes,verbatim"]
----
# yum update
# cp /usr/share/ansible/roles/oVirt.v2v-conversion-host/files/virt-v2v-wrapper.py /usr/bin/
----

* OpenStack Platform: Download the latest conversion appliance and create new conversion hosts.
====
