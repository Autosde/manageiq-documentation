You can configure your environment for SSH transformation if you cannot use VDDK.

Configuring your environment involves the following additional steps:

. Configuring the VMware hypervisors for SSH transformation
ifdef::rhv_1-1,rhv_1-2[]
. (Optional) Reinstalling `ipa-client` if you are configuring your conversion hosts for SSSD with single sign-on
endif::rhv_1-1,rhv_1-2[]
. Configuring the conversion hosts for SSH
. Copying the VMware keys to the conversion hosts
. Configuring secure remote login to the VMware hypervisors
ifdef::rhv_1-1,rhv_1-2[]
. Authenticating the conversion hosts in CloudForms
endif::rhv_1-1,rhv_1-2[]

== Configuring the conversion hosts for SSH transformation

Follow the procedures in xref:Configuring_the_{context}_conversion_hosts_for_vddk[] with the following changes:

* `extra_vars.yml` file:
+
Specify `ssh` for the `v2v_transport_methods` parameter in `extra_vars.yml`.

* `secure_vars.yml` file:
+
Copy the SSH private key for connecting to the VMware hypervisors for the `v2v_ssh_private_key` parameter in `secure_vars.yml`.
